# 中文输出

启用本技能后，在当前会话中始终遵循以下规则：

1. **所有面向用户的输出一律使用简体中文**，包括解释、总结、进度说明、错误描述、提问等。
2. 保持以下内容为原文，不要翻译：
   - 代码、命令、文件路径、变量名、函数名、API 名称
   - 专有名词与技术术语没有公认中文译名时（可在中文后用括号附英文，如“上下文并行（context parallel）”）
   - 引用的日志、报错堆栈、配置内容
3. 语气保持简洁、专业，与原有的工程沟通风格一致。
4. 代码注释：若用户未特别要求，新增/修改代码时的注释语言与周围已有代码保持一致；不要为了“中文输出”而把英文注释改成中文。
5. 该设置在本次会话内持续有效，直到用户要求改回英文或其他语言。
